## The Mock Listings Dataset

`data/listings.json` contains 40 mock secondhand listings across categories (tops, bottoms, outerwear, shoes, accessories) and styles (vintage, y2k, grunge, cottagecore, streetwear, and more).

Each listing has: `id`, `title`, `description`, `category`, `style_tags`, `size`, `condition`, `price`, `colors`, `brand`, and `platform`.

Load it with:
```python
from utils.data_loader import load_listings
listings = load_listings()
```

## The Wardrobe Schema

`data/wardrobe_schema.json` defines the format your agent uses to represent a user's existing wardrobe. It includes:

- `schema`: field definitions for a wardrobe item
- `example_wardrobe`: a sample wardrobe with 10 items you can use for testing
- `empty_wardrobe`: a starting template for a new user

Load an example wardrobe with:

## Tool Inventory

Your README submission must document each tool's name, inputs, and return value. **These must exactly match your actual function signatures in `tools.py`.** Your documented interfaces will be checked against your actual function signatures in `tools.py` — if the parameter count or types contradict what's in the code, you may not receive full credit for that tool.

### search_listings
- **Purpose:** Filters and ranks the mock listings dataset to find the best matches for what the user described.
- **Inputs:** `description` (str), `size` (str or None), `max_price` (float or None)
- **Output:** List of up to 3 listing dicts ranked by keyword relevance. Each dict has: id, title, description, category, style_tags, size, condition, price, colors, brand, platform. Returns an empty list if nothing matches.

### suggest_outfit
- **Purpose:** Uses the LLM to suggest an outfit combining the new item with pieces from the user's wardrobe. Falls back to general styling advice if the wardrobe is empty.
- **Inputs:** `new_item` (dict), `wardrobe` (dict)
- **Output:** A string with a 2–3 sentence outfit suggestion.

### create_fit_card
- **Purpose:** Formats the outfit and item details into a structured fit card with a social media caption generated by the LLM.
- **Inputs:** `outfit` (str), `new_item` (dict)
- **Output:** A formatted string with four labeled sections — Item, Condition, Styling Tip, and Caption.

## Planning Loop and State

The agent parses the query with regex (price and size), calls search_listings, then checks if search_results is empty. If yes, it sets session["error"] and stops — suggest_outfit and create_fit_card never run. If there are results, it picks results[0], calls suggest_outfit, then create_fit_card in order.

State is a session dict. The dict from search_listings is stored as selected_item and passed directly into the next two tools. The outfit string from suggest_outfit is stored and passed straight into create_fit_card. The user never re-enters anything between steps.

## Error Handling and Fail Points

<!-- For each tool, describe the specific failure mode and what your agent does in response.
     This maps to the error handling section of the rubric (F5-C1). -->

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| `search_listings` | No listings match the query | Sets `session["error"]` = "No listings found. Try a different description, size, or price range." Returns early — the other two tools never run. Example: querying "designer ballgown size XXS under $5" returns `[]` and the agent stops. |
| `suggest_outfit` | Wardrobe is empty | Passes the empty wardrobe to the LLM with a note. Returns general styling advice for the item on its own, e.g. "This tee pairs well with wide-leg jeans for a vintage look." Never crashes. |
| `create_fit_card` | `outfit` string is empty | Returns `"Error: outfit data is incomplete — no fit card generated."` without calling the LLM. Confirmed by running `create_fit_card("", item)` directly. |

## Extra Features

### Price Comparison
compare_price(new_item) compares the item's price to other listings in the same category from the dataset. No LLM needed. Returns a short verdict like "good deal" or "above average." Shows up at the bottom of the listing panel.

### Retry Logic
If search returns nothing, the agent retries with size and price filters removed. If that finds results it keeps going and shows a note saying what was adjusted. If still empty, it sets the error and stops.
